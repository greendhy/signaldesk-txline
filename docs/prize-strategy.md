# TxLINE Trading Tools Prize Strategy

## Objective

Win a top-three prize in the TxODDS/Superteam "Trading Tools and Agents" World Cup hackathon by building a working, judge-testable tool that makes TxLINE look commercially valuable, not just technically connected.

## Deadline

- Submission closes: July 19, 2026 23:59 UTC.
- Beijing time: July 20, 2026 07:59.
- Winner announcement: July 29, 2026 15:00 UTC.

## Cost Policy

Default path: no paid services.

Allowed without stopping:

- Local development.
- Public GitHub repository setup if already authenticated.
- Free-tier deployment.
- Devnet-only blockchain work.

Stop and ask the user before:

- Any wallet signature.
- Any mainnet transaction.
- Any service asking for a card or paid plan.
- Any upload/submission that cannot be undone.

## What Judges Explicitly Want

- Working running agent or tool.
- TxLINE live feed as an input.
- Autonomous decisions after deployment.
- Clear, deterministic, documented strategy logic.
- Production-readiness for a professional trading team, market operator, or B2B intermediary.
- Strong demo video, because live match activity may be unavailable during review.
- Public repo, accessible MVP/API, short technical docs, and API feedback.

## Winning Angle

Build a verifiable trading operations workbench:

- Live/replay TxLINE ingestion.
- Strategy agents that react to odds and score changes.
- Risk controls and paper execution.
- Decision receipts that preserve inputs, timestamps, thresholds, and proof metadata.
- Replay mode for judge demos when no match is live.
- A professional dashboard that makes TxLINE's data layer feel deployable by a trading desk or market operator.

## Avoid

- A simple odds alert bot.
- A UI-only dashboard with no autonomous execution.
- A black-box LLM agent with vague decisions.
- A betting product that looks legally reckless.
- A submission that requires judges to create wallets, pay fees, or configure complex services.

## Product Name Candidates

- SignalDesk
- TxLINE Command Center
- ProofTrader
- Verifiable MarketOps

Working name for now: SignalDesk.

## Required Deliverables

- Public repo.
- Running app or functional API endpoint.
- Up to 5-minute demo video.
- Technical documentation listing TxLINE endpoints used.
- Submission answers and API feedback.

## User Intervention Gates

The user should only be stopped for:

- Wallet connection/signature to activate TxLINE API access.
- External account login if deployment or GitHub auth is missing.
- Final review and Superteam submission.
- Demo video recording or approval.
